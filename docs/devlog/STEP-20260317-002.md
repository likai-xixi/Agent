## Risks

1. OSS leader election depends on object-store availability and reasonably synchronized lease timing.
2. Mitigation: satellite fallback is automatic on lease errors, IM ingress is only accepted on elected master, and lease expiry enables failover instead of split-brain overwrite.
3. Hard budget enforcement can halt provider execution if operators forget to replenish balance state.
4. Mitigation: `/health` and `/ops/budget` expose remaining balance and daily spend, and the rejection path is explicit with `BALANCE_EXHAUSTED` or `DAILY_BUDGET_EXCEEDED`.
5. Stricter path truth validation may reject workflows that previously relied on implicit directory creation or silent overwrite.
6. Mitigation: write/move/delete behavior is now deterministic, preflight authorization remains intact, and interrupted local mutations still recover through the step journal.
