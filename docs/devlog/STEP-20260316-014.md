## Risks

1. `JsonFileTakeoverStore` writes whole JSON document on each update; high-frequency environments may face write contention.
2. Mitigation: abstraction allows swapping to DB-backed store in phase-2 without API changes.
3. File-based store does not provide transactional cross-process locking.
4. Mitigation: keep single-writer deployment pattern for v1 and move to transactional backend for multi-instance.
