## Risks

1. Local runner auto-snapshot strategy creates additional Git commits before sensitive mutations.
2. Mitigation: snapshots are trace-tagged, resumable work is journaled in `steps.jsonl`, and rollback commands are explicit in the checkpoint metadata.
3. Dynamic MCP mounting can execute third-party server code if approved carelessly.
4. Mitigation: only `npx`-based mounts are accepted, mount requests require explicit approval, and mounts are persisted in a local registry for auditability.
5. Semantic cache and knowledge transfer are local-only but still store prompts/results on disk.
6. Mitigation: storage is under the portable data root, sensitive prompt material is scrubbed before outbound provider calls, and cache files can be backed up or moved separately from source code.
